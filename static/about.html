About Multithreaded Web Server

Project Overview

Multithreaded Web Server is implemented in Python. It is designed to handle multiple client requests simultaneously, serving static files from the server's file system. This lightweight server provides the essential functions of a basic web server, allowing users to host HTML pages, CSS, JavaScript, and other static files.

Key Features

  • Concurrent Client Handling: Efficiently handles multiple client connections using threads, supporting high concurrency.
  • Static File Serving: Serves HTML, CSS, JavaScript, images, and text files directly to clients.
  • Basic HTTP Support: Handles simple HTTP GET requests, enabling clients to retrieve server resources.
  • Request Logging: Logs client IP, requested resource, and response status to a log file, providing an access history.
  • Error Handling: Responds with appropriate HTTP error codes for missing or restricted files, including 404 (Not Found) and 500 (Internal Server Error).
  • Resource Synchronization: Ensures thread-safe access to shared resources, preventing data conflicts.
  • Dynamic Content: Generates dynamic content such as a digital clock and current date/time via a REST API or a custom HTML page.

How to Use

To use the server, simply place HTML and other static files in the static directory. The server will automatically serve files from this directory based on the client’s request URL.

To access the server from your browser, enter the IP address and port in the following format: 

http://127.0.0.1:8080/index.html

Replace 127.0.0.1 with your server’s IP address if connecting from a different device on the same network.
